# Privacy Policy – Robot Management System

## 1. Introduction
This document describes how the Robot Management System handles personal and operational data.  
The system is designed to monitor and control a remote robot through a web dashboard. Because the system includes user authentication and operational logging, it may process certain personal data.

This policy ensures that the system follows good legal, ethical, and professional practices when handling data.

---

## 2. Personal Data Collected
The system may collect limited personal data necessary for secure operation:

- Username
- Password (stored in hashed form)
- User role (e.g., Viewer or Commander)
- Command activity logs associated with user accounts

The system does **not** collect unnecessary personal data such as location, biometric data, or personal identifiers beyond what is required for authentication.

---

## 3. Operational Data Collected
The system also collects non-personal operational data related to the robot:

- Robot telemetry (battery level, position, status)
- Commands issued to the robot
- System timestamps
- Connection status

Operational data is required for monitoring robot performance and ensuring safe operation.

---

## 4. Purpose of Data Collection
Data is collected for the following purposes:

- Authenticating users before allowing robot control
- Enforcing role-based access control
- Recording command history for safety auditing
- Monitoring robot health and status

These functions ensure safe and responsible use of the robot system.

---

## 5. Data Security
To protect user data, the system implements several security practices:

- Passwords stored using secure hashing
- Access control through role-based permissions
- Logging of robot commands for accountability
- Restricted access to the robot control interface

Future improvements may include encrypted connections (HTTPS) and secure token-based authentication.

---

## 6. Data Retention
Operational logs may be retained for system diagnostics and safety auditing.  
Personal data will only be retained as long as required for system operation.

---

## 7. Ethical Considerations
Robotic systems have the potential to cause harm if misused.  
For this reason, command logging and user authentication are implemented to ensure accountability and prevent unauthorized control.

The system is designed to promote safe, responsible, and transparent use of robotic technology.

---

## 8. Contact
For questions regarding data usage or system privacy practices, the system administrator should be contacted.

## 9. Legal Framework

This privacy policy is informed by the following legal and professional frameworks:

- UK General Data Protection Regulation (UK GDPR)
- Data Protection Act 2018
- Computer Misuse Act 1990
- ISO/IEC 27001 Information Security principles

These frameworks provide guidance on lawful data processing, system security, and responsible access to computer systems.
